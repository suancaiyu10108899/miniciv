# GDD — 棋盘、地形与地图生成

## 棋盘

| 维度 | 决策 | 设计意图 |
|---|---|---|
| 形状 | 方格 + 四邻 | 简单清晰。四邻让地形障碍更"硬"，创造战略要道 |
| 边缘 | **环面（torus）** | 左右上下均联通。消除角落安全——任何位置在拓扑上等价，360°防守 |
| 尺寸 | **15 / 30 / 50**，默认 30 | 限定档位→平衡性只需测试 3 种 |
| 种子 | 伪随机，u64 | 等种子 = 等地图，评估可复现 |

### 环面距离计算

```python
def torus_delta(a, b, size):
    d = b - a
    if d > size // 2: d -= size
    elif d < -size // 2: d += size
    return d

dist = abs(torus_delta(x1, x2, W)) + abs(torus_delta(y1, y2, H))
```

## 地形

| 地形 | 比例(balanced) | 可建设施 | 通行 | DEF | 弹道 | 视野 |
|---|---|---|---|---|---|---|
| 平原 | 35% | 农场(粮) | 全员 | 0 | 穿透 | 穿透 |
| 森林 | 28% | 伐木场(木) | 全员。骑兵遇林停 | +10 | **阻断** | **阻断** |
| 山地 | 22% | 矿山(金) | 步/弓/侦察。骑兵**禁入** | +15 | **阻断** | **阻断** |
| 水域 | 8% | 无 | **无人可通过** | — | 穿透(弓跨1格) | 穿透 |
| 城市 | 2格 | — | 己方全员 / 敌方近战=攻击 | +25 | — | — |

### 设计意图

- 水域 = 绝对移动障碍→环面下产生走廊和隘口。但水域**不阻断弹道和视野**（弓可跨 1 格水射，侦察兵可看过去）
- 森林/山地阻断弹道→创造掩体和制高点
- 水域生成约束：城市四正方向不得为水域（保证出入）。BFS 验证双方可达。水域不形成全封闭环
- 设施建在对应地形（产出来自设施而非地形本身，见 `economy.md`）

## 地图生成器

独立模块。输入 `(seed, size, generator_id)` → 输出 `Map`。

### 算法骨架

```
1. 全平原初始化
2. 按生成器规则放置地形种子 → BFS 聚类扩散
3. 调整比例至目标值
4. BFS 连通性验证（P0 城市→P1 城市可到达）
5. 验证 P0/P1 城市四正方向 = 平原
6. 不通过→换 seed 重试
```

### 生成器列表

| ID | 配比(平/林/山/水) | 聚类 | 用途 |
|---|---|---|---|
| `balanced` | 35/28/22/8 | 中。林 3-8 格/簇，山 2-5 格/簇，水 3-6/湖 | **默认** |
| `symmetric` | 同 balanced | 同。双方半图镜像 | AI 纯策略对比 |
| `fertile` | 45/30/15/3 | 弱。大平原，少量小山包 | 资源充裕 |
| `harsh` | 25/25/30/15 | 强。山脉成脊，平原碎片化 | 强制冲突 |
| `mountain_pass` | 30/25/30/8 | 山沿 2-3 条脊集中，通道 1-3 格宽 | 隘口战术 |
| `archipelago` | 30/28/18/20 | 水 20%，分割为"岛"，岛间 1-2 格通道 | 环面群岛 |

## 开局位置

- P0 城市随机（BFS 连通区内）
- P1 城市 = P0 + `(W/2, H/2)` mod (W,H)
- 30×30 上距离 = 15+15 = 30 格
