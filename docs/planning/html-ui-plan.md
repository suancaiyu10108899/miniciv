# HTML UI 规划

> 原型验收阶段的交互界面。目标：浏览器里看到游戏、操作单位、看回放。

## 设计原则

- 零依赖。单个 HTML 文件，内嵌 CSS+JS，浏览器打开即用
- 规则引擎不嵌入 JS。HTML 通过 HTTP server 调用 Python 后端，或者：生成静态 JSON 快照序列，JS 播放
- 回放和实时对战复用地图像素渲染逻辑

## 两种模式

### 模式 A：实时对战（Human vs AI）

```
浏览器 ← HTTP → Python 后端(规则引擎)
  1. 浏览器发当前状态 → Python 渲染 HTML
  2. 人类在浏览器操作 → 提交动作 → Python 处理 → 返回新状态
```

需要 Flask/FastAPI。多一个依赖。

### 模式 B：静态回放 + 逐步对战

```
Python 跑完整一局 → 生成 JSON 快照序列 → 浏览器播放
  1. 人类每回合操作 → Python 实时处理 → 保存快照
  2. 浏览器加载快照序列 → 逐帧可进可退
```

### 决策：用模式 B 的变体

- **debug 用**：Python 生成一局完整 JSON → HTML 回放（种子+动作序列 → 重执行 → 快照序列）
- **验收用**：改进 play.py → Python 终端交互更友好（非阻塞管道），HTML 仅做回放

HTML 先做回放——这是 AI 调试和验收的核心需求（看清楚 AI 到底在干什么）。实时交互式对战以后再迭代。

## 文件规划

- `prototype/render_html.py` — Python 后端：生成回放 JSON + HTML 页面
- `prototype/static/replay_viewer.html` — 前端：加载 JSON → 彩色格子 + 视角切换 + 逐帧

## 功能列表

- [ ] 30×30 彩色格子（地形色 + 单位图标 + 设施标记 + 迷雾）
- [ ] 上一回合/下一回合/自动播放 + 跳转
- [ ] P0 / P1 / 上帝三个视角
- [ ] 双方资源面板 + 科技树可视化
- [ ] 单位点击→显示属性
- [ ] 终局提示

## 不做(v1)

- 浏览器里的实时操作（终端 play.py 够了）
- 移动范围高亮
- 音效/动画
