# ADR-002：无单位数量上限

## 决策

不设单位数量上限。AI 自己决定造多少兵/工人。

## 理由

1. **AI 决策维度。** 上限 = 剥夺了"爆兵还是建设"的选择。无上限 → AI 必须学会权衡
2. **经济自约束。** 维护费（待测试）+ 资源有限 + 生产格位有限（四邻）→ 自然限制，不需要硬编码
3. **旧项目教训。** 固定 2-3 单位/方 → 单位死后自动补 → AI 不需要珍惜单位。无上限 + 每单位有资源成本 → 单位是资产

## 风险

- 无维护费时最优策略 = 无限爆兵。需要原型测试确认维护费是否必要
- 如果维护费太低 → 爆兵碾压一切 → 建设胜利不可能
