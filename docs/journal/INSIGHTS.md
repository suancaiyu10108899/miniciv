## Insight 列表

### I001: Facility=8 完全不可达——门槛悬崖效应

**日期**: 2026-07-03
**来源**: Facility=8 验证实验（3,200 局）

**现象**: 3,200 局中 0 次建设胜利。最好的是 Evo 平均建 4.5 个设施，离门槛 8 差将近一倍。同时 Evo 依然 75% 胜率——通过 construction_count 在阶梯判定中碾压。

**反直觉之处**: 修复前 Evo 42 回合 C5 rush 赢（建设胜利过强）。修复后建设胜利不存在，但 Evo 胜率基本没变——它只是换了个赢法。调了一个参数，问题从一个维度转移到另一个维度了。

**解释**: 建设胜利是二元的（到达门槛就赢，不到就零），而 construction_count 是连续的（每多一个 C 线科技就多一分阶梯判定优势）。Evo 在冲到 C4 之后自然地积累了 4-5 个设施和 4.4 个 C 线科技——不够赢 via 建设，但足够赢 via 阶梯。

**启示**: 
1. 二元门槛在策略游戏中天然危险——太低了 rush，太高了不可达，中间地带很窄。
2. 当一种胜利路径被堵死，AI 不会"尝试另一种"——它会找到同一个目标函数下的另一个最优解。
3. 调参的时候要同时看"胜率"和"获胜方式"——胜率不变不代表参数没影响。

**关联**: `experiments/v0.5.0/facility-8-verify/`、`docs/planning/2026-07-03-wrapup.md`
