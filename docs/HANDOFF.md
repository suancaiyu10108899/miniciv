   修完后军事从假象变真实,重找甜点 = 成本×2(三方制衡)。

**两次差点把假象当胜利,都被诊断/回放揭穿**(资源甜点假象、Defender speedrun)。没重蹈前两个 AI"过早宣布胜利"。

## 下一步:深化游戏内容(根本性深度)

深度"温和"的根源 = 内容维度少(单城、5兵种、无迷雾、无科技强互斥)。深化 = 增加决策维度。
**建议顺序(深度收益/成本)**:
1. **科技强互斥**(最便宜):让 M/E/C 三线抢有限研究槽,或路线真互斥 → 逼出路线选择。
2. **迷雾 FOW**(中):信息博弈,侦察兵终于有用。Rust 需实现可见性(改 Agent 接口)。
3. **多城/开拓者**(大):空间博弈。

**铁律:每个深化都用 `bin/depth`(决策分叉/应变)+ `bin/table`(支配矩阵)体检,确认它真加了深度(黄灯转绿),不是加噪声。**

## 默认参数改不改 —— 我的思路(留给你判断)

**现状**:`GameConfig::default()` 还是历史基线(成本×1 = 5T 速通旧参数);新甜点成本×2 只记在文档/评估参数里。
所以跑默认 `eval`/`table`/`replay` 看到的是**速通支配**,不是甜点——要传 `25 2.0 160` 才是甜点。

**选项A:设默认 = 甜点(成本×2, HP160)**
- 优点:默认就是健康游戏,不用每次传参。
- 代价:要更新一批测试(反速通哨兵断言"Builder 5T速通"、config 测试断言默认5T);重新生成 `replays/`。

**选项B:不改默认,文档记录甜点参数**
- 优点:省事。
- 缺点:默认跑看到速通,得记得传参数。

**我倾向 B,但判据交给你**:
- 若你**马上深化**(改科技树/加迷雾会再次改变甜点)→ 现在锁死默认无意义,选 B。
- 若你想**先稳定/发布一个"一阶深度完成版"**(打 tag、作基线)→ 选 A(设默认+更新测试+反速通哨兵改为"守护≥30T"),把这一版固化。

关键:哨兵测试(`ai/fixed.rs` `test_反速通哨兵`)现在守的是"Builder 5T速通不健康基线",注释里写了"打破后要改为 ≥30T"。设默认=甜点时,记得同步更新它。

## 方法论铁律(踩过坑总结)

1. **参数只设"环境",平滑平衡靠 AI 决策**(别调血量到 50%;血量阶跃是攻击次数离散的正确反映)。
2. **探针数据方向可信、精度不可信**(骑兵绕路修复后数字变但方向稳)。
3. **小样本别下结论**(30-seed 的"P0偏差63%"是噪声,500-seed 是 50%)。
4. **看回放 + 代码审查机制正确性,数字工具不可替代**(攻城bug/兵种残缺,胜率数字看不出)。
5. **别过早宣布胜利**——两次假象都靠 turn分布/回放揭穿。
6. **真相活在可执行工件里**(eval/depth/探针/哨兵),不活在对话上下文。
