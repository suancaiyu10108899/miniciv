# 平衡调优日志

## 初始状态（07-01 晚）

CITY_HP=150, CITY_DEF=10, 设施产出=1(bug), 初始15/15/15
结果: 100% tiebreak, 22T, 城市无法攻破

## 迭代记录

### 第一轮：城市占领机制修复
- 修复: 占领一次性判定→渐进伤害, 空城占领, 交替先手
- CITY_HP: 150→100
- 地形DEF: forest 10→5, mountain 15→8, city 25→15
- C2: city_hp+100→+30
- 结果: 100% conquest, 22T, P0=49%

### 第二轮：城市防守 + AI 守城
- CITY_DAMAGE=20 (城市对占领者造成伤害)
- Greedy/Aggressive 添加守城逻辑
- 结果: 60T 消耗战, 95% conquest, 4% tiebreak, 首次出现建设胜利幽灵

### 第三轮：建设胜利路径
- C线成本降40%: C1(6,6,3)→(5,5,3), C5(12,12,12)→(4,4,4)
- 单位成本降30-40%
- 初始资源15→25
- 研究优先于生产
- 结果: Greedy vs Random 65% 建设率

### 第四轮：经济加速 + 攻击力提升
- 全部兵种 ATK+5
- CITY_DAMAGE 20→15
- 设施产出 3→4
- C线成本再降: C5(4,4,4)→(2,2,2)→回调至(3,3,3)
- 研究tick移至双方同时
- 结果: 当前最优参数

### 第五轮：参数网格扫描确认
- 9组合×900局=8100局
- 确认 HP=100 DMG=15 为最优点
- P0基线52%(最接近50%), 建设率64%, 综合tie 17%

## 当前参数（冻结）

```
CITY_HP=100, CITY_DEF=10, CITY_DAMAGE=15
设施产出=4, 初始资源=25/25/25
步兵 ATK=45/DEF=30, 骑兵 ATK=60/DEF=15
交替先手(奇P0先/偶P1先), 研究双方同时tick
C1(5,5,3) C3(5,5,5) C4(6,3,3) C5(3,3,3)
tiebreak 随机判定
```

## 关键经验

1. 参数网格扫描是最高效的调优手段
2. 多个参数同时调会产生不可预知的交互
3. Aggressive 镜像平局不是参数问题——所有9组参数下都是76-89%
4. tiebreak 默认 P0 胜是隐藏的系统性偏差
