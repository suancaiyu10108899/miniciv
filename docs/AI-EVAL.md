# AI 评估标准化协议

> 目标：区分"训练策略不行"和"实现有 bug"。每个 AI 必须通过三级评估才能宣称有效。

---

## 一、三级评估协议

### L1: Sanity Check

**命令**:
```bash
python -m prototype.eval --ai0 <ai> --ai1 random --games 500 --size 15
```

**通过标准**: 胜率 > 70%（非随机基线的最低门槛）
**失败意味着**: AI 有严重 bug，或者策略比随机还差（如 BC vs Random = 0%）

### L2: Baseline

**命令**:
```bash
python -m prototype.eval --ai0 <ai> --ai1 greedy --games 500 --size 15
```

**通过标准**: 胜率 > 30%（至少能对 Greedy 产生一些威胁）
**失败意味着**: AI 太弱，连手写基线都打不过，不值得进入全矩阵

### L3: Full Matrix

**前提**: L1 + L2 通过
**做法**: 加入 7×7 或 N×N 全矩阵 paired 评估
**通过标准**: Elo 排名不是最后一名

## 二、训练诊断 Checklist

每个学习型 AI 在"跑→看胜率"之前必须检查：

### 必检项

| # | 检查项 | 方法 | 正常范围 | 异常意味着 |
|---|--------|------|---------|-----------|
| 1 | 权重范围 | `np.isfinite(weights).all()` | 全部有限 | NaN/Inf → 学习率太高/梯度爆炸 |
| 2 | Loss 趋势 | 记录每 N 局的 loss | 单调下降或平台 | 不降 → reward 设计问题/网络太浅 |
| 3 | 策略多样性 | action distribution entropy | entropy > 0（不总选同一个动作） | entropy=0 → 策略坍塌 |
| 4 | 对手差异化 | 对 Random vs Greedy 的行为差异 | 对 Greedy 的 action distribution 显著不同 | 相同 → AI 不关心对手在做什么 |

### 建议检项

| # | 检查项 | 方法 | 正常范围 |
|---|--------|------|---------|
| 5 | 训练速度 | games/sec | 取决于 AI，记录基线 |
| 6 | 胜率曲线 | 每 N 代的 vs Random 胜率 | 单调上升（除 BC 外） |
| 7 | 对战类型分布 | 征服/建设/阶梯的比例 | 应该变化（说明策略在演化） |

## 三、AI 健康报告模板

每个 AI 必须有这样一份报告：

```markdown
## AI: <name>

### 基本信息
- 类型: [手写规则 | 进化权重 | 行为克隆 | 深度RL | 混合]
- 参数数量: N
- 训练数据/代际: X

### L1: Sanity Check
- vs Random 胜率: XX%
- 通过: [是/否]

### L2: Baseline
- vs Greedy 胜率: XX%
- 通过: [是/否]

### 诊断结果
- 权重: [正常/NaN/未检查]
- Loss 趋势: [下降/平台/上升/不适用]
- 策略多样性: [高/中/低/坍塌]
- 对手差异化: [有/无/不适用]

### 已知问题
- [问题描述]

### 判定
- [有效 | 有bug需修复 | 训练策略无效 | 待验证]
```

## 四、评估优先级

1. **Evo** — 当前最强，确认没有过拟合
2. **DQN** — 已知 NaN bug，最需要诊断
3. **FlatMC** — 确认是参数问题还是方法问题
4. **SelfPlay** — 三代无改善，需要决策继续还是放弃
5. **BC** — 确认泛化失败是否是固有问题
6. **Hybrid** — 确认最弱原因
