# miniciv 当前状态完整分析 — 2026-07-02

## 一、游戏机制

### 地图与棋盘
- 15x15 环面棋盘 (wrap-around)
- 6种地形生成器: balanced/symmetric/fertile/harsh/mountain_pass/archipelago
- 地形比例(balanced): 平原35% 森林28% 山地22% 水域8% + 2城市格
- 环面意味着单位可以穿越边界——到达敌城距离约7-8格

### 兵种与战斗
- 5兵种: infantry(HP100/ATK40/DEF30/MOVE1), cavalry(80/55/15/2), archer(60/45/10/1), scout(40/10/5/2), worker(10/0/0/1)
- 战斗公式: damage = max(1, ATK + att_terrain - DEF - def_terrain)
- 骑兵冲锋: 走2格平原后攻击 +10ATK
- 弓箭手远程: 单向输出, 守方不还手, 但不能占领城市
- 地形防御: plain=0, forest=5, mountain=8, water=0, city=15

### 经济系统
- 三资源: 粮食/木材/金币
- 初始: 15/15/15, 3工人+1侦察兵
- 设施产出: 3/回合 (农场=粮食, 锯木厂=木材, 金矿=金币)
- 城市基础: +1粮食/回合
- 单位成本: infantry(8粮), cavalry(8粮+5金), archer(5粮+5木), scout(5粮), worker(5粮)

### 科技树 (13节点DAG)
- 军事线M1-M4: 步兵/弓手ATK+5, 骑兵冲锋+5, 步兵山地森林DEF+10, 全军HP+10
- 经济线E1-E4: 农场+1, 锯木厂+1, 金矿+1, 工人移动+1
- 建设线C1-C5: C1解锁建设, C2城市HP+30, C3研究加速, C4城市粮食+2, C5建设胜利
- 前置逻辑: M4=M2 OR M3, C4=C2 OR C3, E4=E2 OR E3, C5=C3 AND C4

### 胜利条件
- 征服: 敌方城市HP<=0 (单位占领时每回合造成渐进伤害)
- 建设: 研究完成C5科技
- 100回合阶梯判定: construction_count -> city_hp -> P0胜

### 城市占领机制 (本次修复核心)
- CITY_HP=100, CITY_DEF=10
- 步兵占城伤害: max(1, 40-10)=30/回合 → 需要4回合占领
- 骑兵占城伤害: max(1, 55-10)=45/回合 → 需要3回合占领
- 渐进伤害: hp -= dmg (之前是一次性判定,永远打不下)
- 空城可直接走入开始占领 (之前空城走过去不触发)
- 交替先手: 奇数回合P0先, 偶数回合P1先
- CITY_DAMAGE=20: 城市每回合对占领者造成20伤害
  - 骑兵占城3T承受60伤害(HP80→20), 勉强存活
  - 步兵占城4T承受80伤害(HP100→20), 勉强存活
  - 纯rush仍可行但高风险, 需要军队协同
