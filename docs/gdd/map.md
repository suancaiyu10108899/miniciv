# GDD — 棋盘、地形与地图生成

## 棋盘

| 维度 | 决策 | 设计意图 |
|---|---|---|
| 形状 | 方格 + 四邻 | 简单清晰。四邻让地形障碍更"硬"，创造战略要道 |
| 边缘 | **环面（torus）** | 左右上下均联通。消除角落安全——任何位置在拓扑上等价，360° 防守 |
| 尺寸 | **15 / 30 / 50**，默认 30 | 限定档位而非任意值 → 平衡性只需测试 3 种 |
| 种子 | 伪随机，u64 | 等种子 = 等地图，评估可复现 |

### 环面距离计算

```python
def torus_delta(a, b, size):
    d = b - a
    if d > size // 2: d -= size
    elif d < -size // 2: d += size
    return d

dist = abs(torus_delta(x1, x2, W)) + abs(torus_delta(y1, y2, H))
```

## 地形

| 地形 | 比例(balanced) | 产出(工人) | 通行 | 防御加成 | 弹道穿透 |
|---|---|---|---|---|---|
| 平原 | 35% | 粮食 | 全员 | 0 | 是 |
| 森林 | 28% | 木材 | 全员。骑兵遇林停(移速归1) | +10 | **阻断** |
| 山地 | 22% | 金币 | 步/弓/侦察。骑兵**禁入** | +15 | **阻断** |
| 水域 | 8% | 无 | **无人可通过** | — | 是（不阻断弹道） |
| 城市格 | 2(H×W) | — | 己方全员 / 敌方近战（=攻击） | +25 | — |

### 设计意图

- 四种地形创造四种不同的移动/战斗/经济决策
- 水域 = 绝对障碍 → 环面下产生"走廊"和"隘口"——两个湖泊之间的狭窄通道是天然战略要道
- 森林/山阻断弹道 → 地形不仅影响移动，还影响远程战斗。创造"制高点"概念
- 骑兵禁山 + 遇林停 → 骑兵速度快但路线受限。步兵慢但任何地方都能去
- 弓箭射程 + 地形阻断 = 森林后的弓箭手射不到平原上的敌人，但山上的弓箭手可以越过森林射平原（未来 LOS v2 实现）

## 地图生成器

生成器是独立模块。每个生成器接收 `(seed, size)` → 返回 `Map`。

### 算法骨架（所有生成器共用）

```
1. 初始化为全平原
2. 按生成器特定规则放置地形种子 → BFS 扩散
3. 调整比例至目标值
4. BFS 连通性验证（城市 A → 城市 B 可通行）
5. 验证城市 3×3 初始布局完整（四正=平原）
6. 不通过 → 换 seed 重试
```

### 生成器列表

| ID | 配比(平/林/山/水) | 聚类 | 用途 |
|---|---|---|---|
| `balanced` | 35/28/22/8 | 中等。森林 3-8 格/簇，山地 2-5 格/簇，水域 3-6 格/湖 | **默认**。通用 |
| `symmetric` | 同 balanced | 同 balanced。**双方半图镜像** | AI 纯策略对比，消除地形噪声 |
| `fertile` | 45/30/15/3 | 弱。大平原，小山包，很少水 | 资源充裕 → 建设胜利快 |
| `harsh` | 25/25/30/15 | 强。山脉成脊，水域成湖，平原碎片化 | 资源稀缺 → 强制冲突 |
| `mountain_pass` | 30/25/30/8 | 山地沿 2-3 条"脊"集中，通道宽 1-3 格 | 隘口战术价值极高 |
| `archipelago` | 30/28/18/20 | 水域 20%，分割成多个"岛"，岛间 1-2 格通道 | 环面上的群岛——侦察兵关键 |

### 双方开局位置

- P0 城市随机放置（在 BFS 连通区内）
- P1 城市 = P0 坐标 + `(W/2, H/2)` mod (W,H)
- 30×30 环面上距离 = 15+15 = 30 格（骑兵平原全速 ~15T 到）
- 双方初始 3×3 布局镜像（四正平原 + 四角山/林交替）
