# 平衡扫描发现 — M1.2

> 2026-07-10 | 第三个 AI | GameConfig 参数化 + scan 工具
> 复现: `cargo run --release --bin scan -- 200`

## 方法

M1.1/M1.2 把平衡参数变成 `GameConfig`(可运行时配置)。`bin/scan` 对每组参数测两个指标:
- **speedrun_turn**: Builder vs Random 平均结束回合(建设速通多快)
- **rusher_vs_builder**: Rusher vs Builder paired 胜率(军事能否惩罚裸建设)

甜点 = 建设速通 ≈ 军事攻城时间,使二者势均力敌(Rusher 30~70%)。

## 逐杠杆发现

| 杠杆 | 效果 | 结论 |
|------|------|------|
| 设施门槛 4→8 | 速通回合**几乎不变** | **无效**——初始工人建设施不是瓶颈 |
| C5 耗时 2→6 | 速通 5→7T | 天花板低 |
| 学院减半 关掉 | 速通 5→6T | 天花板低 |
| 三者拉满 | 速通最多 10T | **纯建设侧微调够不着** |
| city_hp 80→40 | Rusher 0%→75% | **阶跃开关**(短窗口下攻城是阈值函数,非渐变) |
| **起手资源 25→17** | **速通 5T→30T** | **决定性主杠杆** |

## 决定性结果:单杠杆甜点

只降起手资源(其他全默认),200 seeds:

| 起手资源 | 速通(T) | Rusher 胜率 |
|----------|---------|-------------|
| 25(现状) | 5.1 | 0.0% |
| 21 | 11.4 | 12.2% |
| 19 | 11.7 | 10.2% |
| **18** | **30.5** | **54.2%** ← 势均力敌 |
| **17** | **30.3** | **53.2%** ← 势均力敌 |
| 16 | 44.2 | 86.5% |
| 14 | 45.0 | 86.2% |

## 结论

1. **一阶甜点存在,且在纯数值范围内**——不需要机制改动、不改科技树内容、不加游戏内容。
2. **关键杠杆是起手资源**(控制建设发育速度),不是设施门槛(无效)或科技耗时(天花板低)。
3. **甜点坐标 ≈ 起手资源 17-18**:建设速通 ~30T、Rusher vs Builder ~53%(势均力敌)。
4. **机制解释**(正是目标张力):资源少 → 建设者必须先发展经济才能负担 C 线科技 →
   "发育 vs 直接建设"取舍出现 → 建设慢到 30T → 军事 27T 攻城刚好够得着 → 势均力敌。
