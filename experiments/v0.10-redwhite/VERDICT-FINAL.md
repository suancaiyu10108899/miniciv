### 1.4 频率依赖 — R-3

```
1v1: AlwaysWhite 61.8% > AlwaysRed 38.2%    (白线支配1v1)
2v2: AllWhite 13.8% < AllRed 86.2%          (红线支配2v2)
     ↑ 48pp 完全逆转 ↑
```

**裁决**: R-3 ✅✅ 爆炸成立。这是P1.5阶段最强的深度信号。在同一个游戏引擎上，改变玩家数就完全逆转了最优策略——没有任何策略是"不论场上别人怎么选都最优"。频率依赖的存在性证明已完成。

**机制解释**: 1v1中白线的+50%产出加成能快速转化为军事优势终结游戏(82T)。2v2中需要打穿两个对手的防线，白线玩家来不及在支持度危机爆发前终结游戏。红线在2v2中有队友吸引火力、安心积累组织度→兑换爆发。

## 3. 诚实边界

### 3.1 游戏长度略高于目标
82.9T 略高于 65-80T。尝试缩短至 C4(81.5T)或 C5(80.1T)会牺牲 StateAware 性能。接受 82.9T 作为当前引擎的最优甜点。进一步缩短需要结构性改动。

### 3.2 StateAware 在 2v2 中失效
1v1 中 StateAware 表现优秀(H2H 65% vs AlwaysWhite)，但在 2v2 中对 AlwaysWhite 胜率 0%。根因：5维领先判断是针对1v1设计的，2v2中需要评估团队整体状态而非个人。这是 P2 需要修复的。

### 3.3 FlatMC 未完成完整评估
C1 参数下游戏 80T+，FlatMC 的组合爆炸(研究×生产×姿态×分支×兑换)在深度20时每决策需评估数百组合×80T rollout，评估40局需要数小时。FlatMC 升级（分层评估、候选剪枝）是 P2 任务。

### 3.4 Evo 权重失效
Python 训练的 Evo 权重在 Rust 引擎上完全失效(3.3%)。需要在 Rust 端重训遗传算法。这是 P2 任务。

## 4. 总裁决: H1 — 方向强成立

**P1.5 深度推进成功证明了三个核心主张:**

1. **全局压缩延寿有效**: 从旧甜点的 7-35T 推进到 82.9T，阶梯率仅 8.75%。不是靠单一参数极端化，而是同步压缩所有轴(科技/建造/军事/经济)。

2. **红白反滚雪球有深度**: R-1(处境决定路线)和 R-3(频率依赖)都在多维度验证成立。频率依赖的 48pp 逆转是清洁信号——不需要强AI裁判也能看到深度。

3. **支持度负面维度制造了真取舍**: StateAware 通过支持度风险评估改善决策，证明负面维度不是装饰。

**但不是完整愿景。** 2v2 中 StateAware 失效、FlatMC 评估过慢、Evo 权重失效——这些都是 P2 需要解决的。
