# Game Design Document — miniciv

> 状态：草稿。每个模块逐一讨论定稿。

---

## 设计原则

1. **每一条规则都有设计意图**——它创造了什么类型的决策？如何验证？
2. **AI 友好优先**——规则复杂度 > 美术复杂度。每个机制都要考虑"AI 能否学会利用它"。
3. **先手平衡**——Random vs Random 目标 P0 ≤ 55%。
4. **深度来自组合**——单条规则保持简单，复杂度来自规则之间的交互。

---

## 一、棋盘

- 形状？尺寸？
- 地形类型？各自效果？
- 生成算法？连通性保证？对称性保证？

## 二、单位

- 兵种数量？各自属性差异？
- 这些差异能否创造可区分的策略？
- 生产/招募机制？

## 三、科技

- 结构（树/DAG/线行）？节点数量？
- 研究机制？分支互斥？
- 与胜利条件的关联？

## 四、经济

- 资源类型？收入来源？
- 有没有发展 vs 扩张 vs 军事的 tradeoff？

## 五、胜利条件

- 几种方式？各自条件？
- 不同胜利条件之间有没有策略张力？

## 六、先手平衡

- 选择哪个方案（或组合）？
- 目标：Random P0 ≤ 55%
- Python 原型如何验证？
