## 逐项审计

### Evo #3（ai_evo.py + train_evo.py）

**策略指纹**：骑兵海 + 建设 rush。骑兵占战斗单位 41%，几乎不死工人（0.3/game）。
进化选择了最高效的策略组合——骑兵冲锋+10ATK + 堆叠限制（每个格子独立战斗）+ 建设科技加速。

**已知问题**：89.8% 过强。Gen 0 随机权重已有 76% vs DQN——权重范围本身设计良好，但进化后会过度优化某个特定策略。

**改进方向**：降低 cavalry_production 的权重上限（3.0→2.0），增加对手多样性训练。

### DQN #2（ai_dqn.py + train_dqn.py）

**策略指纹**：纯步兵海。17.0 步兵/game，0 骑兵，0 弓手。最便宜的兵种永远先被生产。

**结构性问题**：**生产决策不经过神经网络。** 6 个动作全是战斗移动（向敌推进/向城/攻击/守城/待命/撤退），生产和研究是硬编码的 `for ut in ["infantry","archer","cavalry"]`。神经网络可以学到完美的战斗策略——但它永远只会造步兵。

**改进方向**：把生产选择放进动作空间（21 个动作 = 6 移动 + 3 生产 + 13 科技），或至少把兵种选择加入状态特征。Rust 重构后做 DQN v2。

### Greedy v6（ai_greedy.py）

**策略指纹**：骑弓混合。步兵 1.3 / 骑兵 2.4 / 弓手 2.9——完全从纯步兵转型。
v6 工人经济多样化（农场/伐木场/矿山）+ Agent D 弓手策略 + Agent E 骑兵策略。

**已知问题**：胜率只有 29.1%，低于 Random 的 37.8%。兵种转型增加了战术复杂度但削弱了纯粹的数量优势。Greedy 镜像仍有 83.2% 阶梯率。

**改进方向**：微调兵种比例。当前弓手占比过高（对 DQN 的纯步兵海效果不好）。可能需要"检测对手兵种 → 调整生产比例"的对手自适应。

### Aggressive v3.4（ai_aggressive.py）

**策略指纹**：骑兵 rush + 工人屠杀。骑兵占比 31%，但工人死亡 6.1/game——死得比造得快。
堆叠限制让波浪进攻无法集中兵力。工人被送到前线建新设施但没有战斗单位保护。

**已知问题**：对 Greedy 0%，对 DQN 2%，对 Evo 3.8%。对所有策略型 AI 几乎全败。

**改进方向**：需要"保护工人"逻辑——只在有战斗单位陪同的区域让工人扩展。或者回退到 v3.3 的保守工人策略（不扩张）。

### FlatMC（ai_flatmc.py）

**策略指纹**：≈ Greedy v6 + 3% 搜索优势。兵种组成和 Greedy 几乎相同（差异 <5%）。
生产/研究/建造完全委托 Greedy。只搜索第一个战斗单位的移动方向。

**已知问题**：Python deepcopy 瓶颈（一局 3-5 分钟）。搜索深度只有 10，rollout policy 是 Random——限制了搜索质量。Rust 重构将解决性能问题，但搜索策略本身也需要升级（至少用 Greedy 做 rollout policy）。

**改进方向**：Rust 后：depth 20 + Greedy rollout policy + 搜索所有单位的移动（不只第一个）。

### Random（ai_rulesrandom.py）

**策略指纹**：随机出弓手（4.9/game——全矩阵最高）。因为弓手成本（3粮+3木=6）现在是中档，随机选择生产时经常抽到它。

**作为基线**：参照价值在下降。当游戏复杂到"随机行为几乎必然输"时，Random 作为 L1 sanity check 的区分力降低。可能需要一个"半随机"基线——随机移动但有基础经济逻辑。

### BC（ai_bc.py）

**当前状态**：未完成。预测器（胜负预测）91.7% 准确率——证明游戏状态信息丰富。但完整 BC AI（per-turn 行为克隆）因为数据格式不匹配 train_bc.py 而未完成。

**改进方向**：Rust 后参数稳定时做。用 Evo + DQN + Greedy 混合数据训练，验证"游戏是否有可学习的策略模式"。

## 跨 AI 发现

1. **弓手从 0→4.4/game 来自经济修复，不是 AI 策略修复。** 工人建伐木场 → 木材有产出 → 弓手可负担 → 自然登场。
2. **骑兵占比分化明显**：Evo 41% > Aggressive 31% > Greedy 36% > FlatMC 6% > DQN 0%。兵种选择是当前 AI 之间的主要区分维度。
3. **工人死亡率是策略质量的负指标**：Aggressive 6.1/game（最差 AI）vs Evo 0.3/game（最强 AI）。
4. **FlatMC ≈ Greedy + 3%**——搜索型在 Python 天花板下的增益有限。Rust 后这个差距应该扩大。
