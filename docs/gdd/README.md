# GDD — 设计原则与全局参数

## 设计原则

1. **每一条规则都有设计意图**——它创造了什么类型的决策？如何验证该决策确实产生了策略差异？
2. **AI 友好优先**——规则复杂度 > 美术复杂度。每个机制都要考虑"AI 能否学会利用它"
3. **先手平衡**——Random vs Random 目标 P0 ≤ 55%
4. **深度来自组合**——单条规则保持简单，复杂度来自规则之间的交互
5. **固定伤害，不随机**——AI 能在 rollout 中精确计算战斗结果，干净的信用分配

## 全局参数

| 参数 | 值 | 说明 |
|---|---|---|
| 回合上限 | **100** | 覆盖探索→发育→对峙→冲刺全周期 |
| 棋盘默认尺寸 | **30×30**（可选 15/50） | |
| 棋盘拓扑 | **环面（torus）** | 左右上下均联通 |
| 城市数量 | **1/方**（不可新建） | v1 单城市 |
| 初始单位 | 2 工人 + 2 侦察兵 | 无初始战斗单位 |
