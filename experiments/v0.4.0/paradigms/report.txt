## 3. Rule+Parameter Hybrid AI

### Rule set (10 hardcoded rules)
1. Enemy adjacent -> attack
2. Enemy on my city -> attack immediately
3. HP < retreat_threshold -> retreat toward city
4. Enemy near my city < defense_range -> defend/intercept
5. My units near enemy city > wave_threshold -> push all
6. Stalemate > patience -> switch to construction tech
7. Resources > save_threshold -> research

## 5. Analysis: Which ML Paradigms Work on This Platform?

### What works well
- **Rule-based systems** (Greedy, Aggressive): Well-understood domain logic translates directly to effective play
- **Evolutionary parameter tuning** (Hybrid): Small parameter spaces are cheap to optimize; rules ensure reasonable play at all points
- **Deterministic heuristics**: Pathfinding, target selection, and resource management are well-served by hand-coded logic

### What is challenging
- **Deep RL (DQN)**: High variance per episode (map generation, combat RNG). Per-step credit assignment is difficult. The action space is per-unit, making state-action mapping ambiguous.
- **Self-play DQN**: Q-network processes a global state vector without unit-level discrimination. Self-play benefits require meaningful representations first.
- **Sparse rewards**: Most game actions have no immediate reward signal, leading to slow learning.

### Lessons for AI-friendly game design
1. **Shape rewards**: Provide intermediate rewards (capturing resources, damaging enemies) to improve learning signal
2. **Reduce action space**: Consider global commands instead of per-unit actions
3. **State representation**: Include spatial features (grid crops near units) rather than global aggregates
4. **Evaluation rigor**: Paired evaluation (swapping P0/P1) is essential to cancel first-move advantage
5. **Domain knowledge injection**: Hybrid approaches (rules + learning) outperform pure learning for this domain's complexity
