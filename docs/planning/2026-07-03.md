## 二、短期收尾（Python 阶段最后一步）

### 方案：设施前置条件 + FlatMC 快速验证

**改动 A**：C5 研究前需 `construction_count >= 3`（实际建了 3 个设施）
- C1 解锁建设能力
- Worker 在 plain→farm, forest→lumbermill, mountain→mine 上建造
- 需要至少 3 个不同地形上的设施 → 迫使 Evo 探索地图 → 延长到 ~50+ 回合

**改动 B**：FlatMC 快速评估（不重训，在线搜索自适应）
- FlatMC vs Evo/Greedy/DQN，各 200 局 paired
- 搜索型方法天然免疫规则变化，零等待验证

**如果达标**（Evo ≤65%，FlatMC/Greedy 有机会反制）→ 参数锁定
**如果不达标** → 继续调 facility 数量或加 AI 拦截逻辑

### 不做的事
- C5 成本倍乘扫描（已证明无效——悬崖太陡）
- 单独调整 C5 turns（Evo 经济可以吸收）
- 全矩阵重跑（太慢，收尾阶段不值得）

## 四、长期：自适应 AI + 自动化（Rust 成熟后）

### 参数条件化训练
- 网络输入层加游戏参数特征（C5 成本、城市 HP、回合上限...）
- 训练时随机采样参数范围
- 一个模型适配整个参数族，规则变更不需要重训

### 种群训练（Population-Based Training）
- 维护一个模型种群，不同个体在不同参数子集上进化
- 定期淘汰差的、突变好的
- 产出不是单一模型，而是模型家族

### Self-Play League
- 历史最佳对手池 + 多样化匹配
- 自动发现游戏平衡问题（联赛统计中"单一策略统治"= 设计问题）
- AI 帮你告诉设计师哪些参数区域值得探索

### 反馈循环加速路径
```
现在: 改参数 → 重训(数小时) → 全矩阵(11h) → 分析 → 再改  = 1-2天/轮
短期: FlatMC快速扫描(20min) → 候选确认(2h) → 只锁定时跑全矩阵 = 2小时/轮
Rust: 参数热力图(<1min) + 在线边测边学 = 分钟级
长期: PBT + League 自动化 = AI 自己迭代
```
