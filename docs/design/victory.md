# GDD — 胜利条件

## 征服胜利

近战单位移动进敌方城市格 → 若伤害 ≥ 城市剩余 HP → 占领 → 立即获胜。

- 弓箭手不能占领（只能削血）
- 己方单位可站在己方城市上防守（享受 DEF+25）

## 建设胜利

完成科技 **C5 纪念碑** = 建设胜利。

C5 需要 C3(学院) + C4(粮仓) 双前置。建设线最少 4 个科技(C1→C2/C3→C4→C5)，总资源花费最大。走建设线意味着牺牲军事 + 经济投入。

## 回合上限

**100 回合**。覆盖探索(0-20T)→发育(15-30T)→对峙(25-70T)→冲刺(70-100T)。

## 阶梯判定

第 100 回合未决出胜负时：

```
1. 完成建设项目多者胜（C1-C5 计数）
2. 同 → 城市剩余 HP 多者胜
3. 同 → P0 胜（500 HP 池下概率 ≈ 0）
```

无平局。RL 每局都有明确胜负信号。

## 设计意图

征服（零和）vs 建设（非零和）：
- 看到对方项目进度在涨 → 必须决策：继续自己建设 or 切军事破坏对方
- 建设线投入资源多 → 军事薄弱 → 对方可能趁虚进攻
- 军事投入多 → 建设慢 → 对方可能比你先完成 C5
- 阶梯判定让双方都有"时间压力"——100 回合不是无限拖延
