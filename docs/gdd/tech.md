# GDD — 科技树

## 结构

DAG（有向无环图）。13 个节点，三线并行。

```
                        ┌─ C2 城墙(+100HP) ─┬─ C4 粮仓(+2粮/T)
C1 奠基 ────────────────┤                   │         │
                        └─ C3 学院(研速÷2) ─┘         └─ C5 纪念碑 ★

E1 耕作(粮+1) ─┬─ E2 伐木(木+1)
               └─ E3 采矿(金+1) ──── E4 物流(工人平原+1移)

M1 锻造(+5ATK) ─┬─ M2 骑兵战术(冲锋+5)
                └─ M3 筑垒(步林/山+10DEF)
                     └─ M4 精兵(+10HP全) ← 需 M2 或 M3
```

## 研究规则

- **开启**：选已有科技的一个**直接后继**，一次性花费资源开启
- **执行**：城市同时只能研究一个科技
- **完成**：1-2T 后完成 → 解锁效果
- **学院(C3)**：完成后研究耗时减半
- **线程不互斥**——可同时拥有三条线的科技。但槽位只有一个

## 信息博弈

| 可见 | 不可见(但可推断) |
|---|---|
| 对方建设项目进度（C1-C5） | 对方战斗科技 → 看单位 ATK/DEF |
| 对方"正在研究中"（但不知研究什么） | 对方经济科技 → 看工人产出速率 |
| | 对方科技树全貌 → 只能推断已完成的 |

## 设计意图

- **C2 vs C3**：第一个大分支。城墙 = 防御 rush，学院 = 投资未来
- **C5 需 C3+C4**：防止建设胜利直冲。最少 4 建设科技 → 大量资源投入 → 必然牺牲军事
- **M2 vs M3**：骑兵路线(机动作战) vs 步兵路线(阵地战)。环面下骑兵绕后价值高，步兵据山更稳
- **E2 vs E3**：木 → 弓手+建设，金 → 骑兵+科技。地图上的森林/山地比例影响最优经济路线
