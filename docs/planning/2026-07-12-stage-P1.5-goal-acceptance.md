## 0. 这是什么

P1.5 是愿景三步走里的**闸门**:在引擎上做**红白反滚雪球核心 + 支持度负面维度**的最小实现,
用裁判 + EGTA + 手玩,裁决"发育状态决定最优路线 + 负面维度取舍"到底有没有深度——
**在投入 P2 完整游戏之前**。**成功 = 诚实知道宝石会不会发光,不是"红白线能赢"。**
H1(有深度)= 赢得建 P2;H0(没深度)= 便宜省下几个月。**两个结果都是胜利。**

## 1. Committed 形态(冻结,别再发散)

从设计轴选项库(愿景 §8.7)里砍定:

- **战力维度**:资源/科技/兵力 **+ 支持度(负面维度)**。← 核心,§8.5。
- **玩家结构**:做成**配置轴**,先开 **1v1 + 2v2**(引擎支持 4 玩家 + 队伍归属)。不硬编码模式。
- **红白线**:两种"支持度取舍管理方式"(白=资本压制+波动;红=拉组织度+跨轴兑换爆发),映射队内分工。
- **空间**:先**单城 + 抽象扩张**(一个"扩张投资"动作:投入换资源、扣支持度);数据不够(取舍无牙)才升多城。
- **协作**:**脚本/隐式**(固定互补角色 + 读棋盘),**不做涌现通信**(MARL 留后)。
- **迷雾/时代/领袖**:**全不做**(留 P2)。
- **评估**:1v1 用现有 paired;2v2 用**队 vs 队** + 小 **EGTA**(策略池混合阵容,量频率依赖)。

### Build 级默认(可 review 时改,别当新设计问题)
- 队伍胜利 = 队内任一人达成胜利条件即全队赢。
- 队友**不共享资源、不共享视野**(无迷雾,视野无意义)。
- 分组、支持度参数全部走 `GameConfig`,默认关(=行为不变),保护现有 80 tests。

## 2. 分级 + 升级触发(便宜先行,数据驱动升级)

```
P1.5-a  1v1 + 支持度取舍(当前2人引擎微扩)   ← 烟雾测试: 取舍有没有牙齿?
        instrument: 支持度取舍是否被行使(agent撞没撞上限/为它调整)?
        ⚠️ 红白在1v1预期平淡——不是kill信号(见§8.6)
P1.5-b  (仅当a"取舍无牙")加多城/空间
P1.5-c  2v2 + 队伍 + 红白映射队内分工 + EGTA   ← 真深度测试
        协作/分工 + 阵容频率依赖在这里才展开
        跑"菜队 vs 强单干"双刃探针
```

**升级触发是数据,不是拍脑袋**:a 的 instrumentation 说"取舍被行使了" → 直接跳 c(2v2);
说"没被行使" → 先 b(加空间)。**别在 1v1 红白上纠缠**(理论已知 1v1 测不出红白真深度)。

---

## 3. 深度规格(验收 = 这些张力是否实现)

**支持度维度(a 阶段):**
- S-1:无视支持度猛拉战力的策略,应输给管理支持度的策略。**若无视支持度也没事 → 负面维度没牙 → 触发升级 b。**
- S-2:instrument 显示支持度取舍**被行使**(agent 会为它调整产军/扩张)。

**红白 + 多人(c 阶段):**
- R-1(反滚雪球核心):领先方选白胜率显著>选红;落后方选红>选白;state-aware 支配 fixed。
- R-2(非反向滚雪球):"故意摆烂→红线暴发"胜率 < 正常发育。
- R-3(频率依赖):EGTA 显示红/白的收益随场上红白比例变化(全白→红渔利;全红→白拍死)。**无策略是"不管别人怎么选都最优"。**
- R-4(协作是深度源):互补角色队(如白引擎+红爆发)> 同质队;"菜队 vs 强单干"探针给出信号。

---

## 4. 止损判据(防"调出新 speedrun"老坑)

- **成功**:S-1/S-2 成立(取舍有牙)+ R-1/R-2/R-3 至少方向成立(红白在 2v2 产生频率依赖)。
- **回退**:在**有界调参**内(支持度惩罚强度、红白兑换率各扫一轮),若 S-1 始终不成立 → "负面维度设计无效";
  若 R-1/R-3 在 2v2 仍不成立 → "红白最小版无深度"。**下 H0 裁决,停,不无限调参。**
- ⚠️ **目标是诚实裁决,不是让红白线赢/好玩。** 中途想把某条线调到碾压 = 掉进 S2 踩过的坑,立刻刹。

---

## 5. 诚实边界(先写死)

- P1.5 用简化白/红 + 脚本协作 + 无迷雾。**NO = "这个最小版没深度",不等于"红白概念死"。YES = 核心健康的有力信号。**
- 1v1 裁"平"必须先查"是不是多人才有的频率依赖没机会展开"(§8.6),再下结论。防假阴性(概念卡`底料太薄导致假阴性`)。
- 2v2 引擎重构是真活(2人硬编码 → 4人+队伍),非微调。这是 P1.5 主要工作量。

## 6. 执行顺序(遵守算力纪律)

```
Step 0: 加 rayon 并行 eval/judge + "并行==串行"确定性测试   (算力纪律, 地基)
Step 1: 引擎玩家结构配置化(支持1v1/2v2, 队伍归属) + 支持度维度 + 抽象扩张
Step 2: 红白分叉(ChooseBranch) + 支持度取舍管理(白压制/红兑换) + dev_lead
Step 3: 探针(AlwaysWhite/Red/StateAware/TankThenRed/互补队/同质队) + FlatMC 适配多人
Step 4: 评估(队vs队 + EGTA 策略池混合阵容) + instrumentation(取舍/频率依赖是否被行使)
Step 5: 分级跑(a→触发→c) + 调参(有界) → 裁决 H0/H1
Step 6: 裁决 → experiments/v0.10-redwhite/VERDICT.md + 文档/记忆/tag/push
```

**开跑任何 2v2/EGTA 矩阵前,按 WORKFLOW 算力调度纪律先审并行**(FlatMC+多人+EGTA 比 S2 更重)。
