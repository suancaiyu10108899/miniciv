# ADR-001：环面拓扑

## 决策

棋盘边缘采用**环面（torus）**——左右上下均联通。否决硬边界和球形。

## 候选方案

| 方案 | 描述 |
|---|---|
| 硬边界（旧项目） | 四边是墙，不可通行 |
| 柱面 | 左右联通，上下不联通 |
| 环面 ✅ | 左右上下均联通 |
| 球形 | 三极坐标球面 |

## 选择环面的理由

1. **消除结构先手优势。** 旧项目硬边界下 P0 左上角开局→后背靠墙→单线推进。环面所有位置拓扑等价，无安全角落
2. **实现简单。** 距离 = `min(|dx|, W-|dx|) + min(|dy|, H-|dy|)`。坐标 wrap = `(x+dx) % W`。无额外数据结构
3. **AI 编码改动小。** 只需把编码中的所有 `(x2-x1)` 换成 `torus_delta(x1,x2,W)`

## 否决球形

球面网格离散化需要足球形铺满（六边形+12个五边形补丁），不同格邻接数不同。几何工程是独立项目级别的工作量。记 backlog，v1 不碰。
