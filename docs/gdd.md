# Game Design Document — miniciv

> 状态：草稿。每个模块逐一讨论定稿。

---

## 设计原则

1. **每一条规则都有设计意图**——它创造了什么类型的决策？如何验证？
2. **AI 友好优先**——规则复杂度 > 美术复杂度。每个机制都要考虑"AI 能否学会利用它"。
3. **先手平衡**——Random vs Random 目标 P0 ≤ 55%。
4. **深度来自组合**——单条规则保持简单，复杂度来自规则之间的交互。

## 一、棋盘

### 已定

| 维度 | 决策 | 设计意图 |
|---|---|---|
| 形状 | 方格 + 四邻 | 简单清晰。四邻让地形障碍更"硬"，创造战略要道 |
| 边缘 | **环面（torus）** | 左右上下均联通。消除角落安全——任何位置在拓扑上等价，360° 防守 |
| 尺寸 | **15 / 30 / 50**，默认 30 | 小尺寸测试用，大尺寸扩展用。限定档位而非任意值→保证平衡性可测试 |
| 种子 | 伪随机，等种子 = 等地图 | 地图可复现。种子空间足够大（u64） |

### 待定（依赖后续模块）

| 维度 | 阻塞模块 | 说明 |
|---|---|---|
| 地形种类与效果 | 单位、经济 | 几种地形？各自对移动/战斗/产出的影响？需要单位兵种差异定了才知道什么地形差异有意义 |
| 连通性保证 | 棋盘（独立） | 强制 BFS 验证——但具体算法等开局位置定了再写 |
| 对称性 | 先手平衡 | 评估时要不要地形对称？等先手方案定了再决策 |
| 视野范围 | 单位移动力 | 和地图大小/兵种移速联动 |
| 开局位置 | 先手平衡 | 等先手平衡方案定了再决定 |

### 不做的

- 球形地图 → backlog。球面网格离散化是独立项目级别的几何问题，v1 不碰

## 二、单位

### 兵种表

| 兵种 | HP | 攻击 | 防御 | 移速 | 视野 | 特殊能力 |
|---|---|---|---|---|---|---|
| 步兵 | 100 | ~40 | ~30 | 1 | 2 | 可上山。林/山 +10/+15 防 |
| 骑兵 | 80 | ~55 | ~15 | 2(平) | 2 | 2 平原→可走 2 格。遇林停。禁山。冲锋 +10 攻 |
| 弓箭手 | 60 | ~45 | ~10 | 1 | 2 | 射程 2。林/山阻断弹道。远程不触发反击 |
| 侦察兵 | 40 | ~10 | ~5 | 2 | **3** | 可上山。无战斗价值，纯信息单位 |
| 工人 | 10 | 0 | 0 | 1 | 2 | 站在对应地形→每回合 +1 资源 |

> ⚠️ 数值是占位符。原型验证时以"几个回合击杀"为目标校准，不以绝对数字为准。

### 战斗规则

- **伤害公式**：`max(1, ATK + 地形攻加成 - DEF - 地形防加成)`，固定值，不随机
- **近战**：双方互打 → 各自扣血
- **远程（弓箭手）**：攻方输出，守方不还手。射程 2，树林/山地阻断弹道。**不能占领城市**
- **击杀回合数目标**：骑兵冲锋 vs 平原步兵 2-3T；步兵山地互殴 6-8T；弓箭 vs 骑兵 2-3T
- **回血**：站在己方城市上每回合 +n HP（具体值原型时调）
- **经验/升级**：不做（回合少，内容少）

### 移动规则

| 兵种 | 规则 |
|---|---|
| 步兵/弓箭手/工人 | 每次 1 格，任何地形（含山） |
| 骑兵 | 2 格平原 → 可走 2 格。第 1 格是森林 → 停。**禁入山地** |
| 侦察兵 | 每次 2 格，**无视地形限制（可上山）** |

### 侦察兵 + 视野遮挡

**v1 先做半径视野**（所有视野范围内格子 = VISIBLE，不管中间地形）。等整套规则跑通，v2 升级为真实 LOS（视线追踪：Bresenham 画线，沿途森林/山地阻断）。LOS 对 AI 编码和 FlatMC rollout 性能有影响（每步需重算可见性），Rust 下不是瓶颈，Python 原型时先用半径视野。

**侦察兵视野**：不考虑 LOS 时，侦察兵的山地站位 = 更好视野（海拔优势）。如果 v2 引入 LOS，侦察兵在山地 → 可越过森林看到远处。但侦擦兵不能有攻击力——站在山上就是个瞭望塔，需要其他兵种保护。

### 克制三角（自然形成，非硬编码）

```
骑兵 ＞ 平原步兵（冲锋 35 伤 → 3 刀）
    ∧
    ╲
步兵山地/森林 ＞ 骑兵（骑兵根本打不了）
    ∧
    ╲
弓箭手 ＞ 骑兵（3 箭杀，骑兵防低）
    ╲
     ∧
步兵山地 ＞ 弓箭手（防高 → 弓箭打不动）
```

### 城市初始布局

3×3 网格环绕城市：

```
山   平   林
平   [城]  平
林   平   山
```

- 四正方向 = 平原（单位自由出入，不被地形卡住）
- 四对角 = 山/林交替（初始三种资源各 2 格可采）

## 三、城市

| 属性 | 值 | 说明 |
|---|---|---|
| HP | 500 | |
| 防御 | 25 | 骑兵(ATK 55) → 30 伤/T。4 骑兵围攻 ≈ 4-5T 破城 |
| 攻击 | 0 | 城市不反击 |

### 占领规则

近战单位移动进城市格 → 计算伤害。若**此移动造成的伤害 ≥ 城市剩余 HP** → 占领成功，征服胜利。弓箭手不能占领城市（只能削血）。

### 生产

单位在城市的**四个正方向邻格**（平原）瞬间生产。生产完成后单位放在邻格空位。四邻格天然限制一回合出兵量（最多 4 个单位/回合，且需要空地）。

### 城市基础产出

城市自身每回合 +1 粮食。工人的产出是增量——即使所有工人被杀，仍有基础恢复能力。

## 四、经济

### 资源

| 资源 | 来源地形 | 采集方式 |
|---|---|---|
| **粮食** | 平原 | 工人站平原 → +1/回合 |
| **木材** | 森林 | 工人站森林 → +1/回合 |
| **金币** | 山地 | 工人站山地 → +1/回合 |

### 花费

| 产出物 | 花费 | 说明 |
|---|---|---|
| 步兵 | 粮食（待定精确值） | |
| 弓箭手 | 粮食 + 木材 | |
| 骑兵 | 粮食 + 金币 | 不同兵种不同资源组合 |
| 侦察兵 | 粮食 | |
| 工人 | 粮食 | 在城市生产 |
| 建设项目 | 粮食 + 木材 + 金币 | 5 个项目各需不同资源配比 |

- **生产机制**：资源足够 → 在城市邻格**瞬间生产** + 当回合可移动。四邻格天然限制一回合出兵量（最多 4 单位，且需空地）
- **开局资源**：每方 5 粮 + 5 木 + 5 金（够造第一个战斗单位但不够连造）
- **开局单位**：2 工人 + 2 侦察兵。无初始战斗单位（自然过渡：冷战争 → 热战争）
- **资源可见**：双方资源池互相可见
- 单位无数量上限。AI 自己决定造多少
- 每单位可能有维护费（待原型测试决定）

## 五、胜利条件

### 征服胜利

近战单位移动进城市格 → 计算伤害。若此伤害 ≥ 城市剩余 HP → 占领成功。弓箭手不能占领。

### 建设胜利

完成 5 个建设项目。每个项目需要不同资源组合。**进度条双方可见**。项目集成在科技树中（建设分支）。

**设计意图**：征服（零和）vs 建设（非零和）之间的策略张力——
- "我自己发展" vs "我阻止对方发展"
- 看到对方进度在涨 → 必须决策：继续建设 or 切军事破坏对方

## 六、科技

### 三分支

| 分支 | 内容 | 信息可见性 |
|---|---|---|
| 战斗 | 单位强化、新兵种解锁、战斗相关 | **不可见**（但可通过对方单位 ATK/DEF 推断） |
| 经济 | 资源效率、工人产出、采集加速 | **不可见**（但可通过对方工人产出推断） |
| 建设 | 5 个建设项目（→ 建设胜利） | **可见**（= 项目建设进度） |

### 研究机制

- **科技图**：DAG（有向无环图），不是纯树。每个科技有一个或多个前置
- **开启**：选已有科技的一个直接后继，一次性花费资源开启研究
- **执行**：城市执行研究（同时只能执行一个科技）
- **完成**：1-2 回合后完成 → 解锁效果
- **分支间**：三条线并行，不互斥——但研究"槽位"只有一个，选了经济就得等经济研究完才能切战斗

### 信息博弈

对方看不到你的战斗/经济科技，但可以推断：
- 对方骑兵 ATK 比你预期的高 → 推断对方有战斗科技
- 对方工人 +2/回合 → 推断对方有经济科技
- 对方建设项目进度在涨 → 直接可见

## 七、迷雾与视野

- **三态**：UNKNOWN（全黑）/ EXPLORED（可见地形，不可见敌方单位）/ VISIBLE（全可见）
- **视野半径**：其他单位 2，侦察兵 3
- **v1 用半径视野**（范围内全部 VISIBLE，不计算地形遮挡）
- **v2 升级 LOS**（Bresenham 视线追踪，森林/山地阻断——和弓箭弹道共用同一套算法）
- 环面下的视野计算：`torus_delta(x1,x2,W) ≤ range`
