# GDD — 迷雾与视野

## 三态迷雾

| 状态 | 能看到 | 变成此状态的条件 |
|---|---|---|
| UNKNOWN | 无 | 从未在任何单位视野范围内 |
| EXPLORED | 地形 | 曾被己方单位视野覆盖过 |
| VISIBLE | 地形 + 单位（敌方+己方） | 当前在己方单位视野范围内 |

## 视野半径

| 兵种 | 视野 |
|---|---|
| 步兵/骑兵/弓箭手/工人 | 2 |
| 侦察兵 | **3** |

## v1：半径视野

视野范围内所有格子 = VISIBLE。不计算地形遮挡（水域不阻断视野）。
环面视野：`torus_delta(x1,x2,W) ≤ range && torus_delta(y1,y2,H) ≤ range`。

## v2：LOS 视线（backlog）

Bresenham 画线算法——从单位到目标格画线，沿途森林/山地阻断视线。
和弓箭弹道共用同一套算法。v2 升级后：
- 侦察兵在山地上 → 可越过森林看到远处（海拔优势、瞭望塔价值）
- 森林后的单位不可见 → 步兵可以"埋伏"
- 环面 + LOS → 信息维度显著复杂 → AI 搜索深度需求增加

v1 先跑通半径视野，v2 再升级。

## 设计意图

- 侦察兵视野 3 vs 其他 2 → 侦察兵 = 信息资产。探索阶段决定权在谁先找到谁
- EXPLORED 保留地形信息 → 你不需要重新探索。但单位会移动 → 必须持续侦察
- 步兵蹲坑 + 侦察兵巡逻 → 维持 VISIBLE 覆盖 → 防止对方从"已探索但无视野"区域偷袭
