# 平衡扫描发现 — M1.2

> 2026-07-10 | 第三个 AI | GameConfig 参数化 + scan 工具
> 复现: `cargo run --release --bin scan -- 200`

## 方法

M1.1/M1.2 把平衡参数变成 `GameConfig`(可运行时配置)。`bin/scan` 对每组参数测两个指标:
- **speedrun_turn**: Builder vs Random 平均结束回合(建设速通多快)
- **rusher_vs_builder**: Rusher vs Builder paired 胜率(军事能否惩罚裸建设)

甜点 = 建设速通 ≈ 军事攻城时间,使二者势均力敌(Rusher 30~70%)。

## 逐杠杆发现

| 杠杆 | 效果 | 结论 |
|------|------|------|
| 设施门槛 4→8 | 速通回合**几乎不变** | **无效**——初始工人建设施不是瓶颈 |
| C5 耗时 2→6 | 速通 5→7T | 天花板低 |
| 学院减半 关掉 | 速通 5→6T | 天花板低 |
| 三者拉满 | 速通最多 10T | **纯建设侧微调够不着** |
| city_hp 80→40 | Rusher 0%→75% | **阶跃开关**(短窗口下攻城是阈值函数,非渐变) |
| **起手资源 25→17** | **速通 5T→30T** | **决定性主杠杆** |

## 决定性结果:单杠杆甜点

只降起手资源(其他全默认),200 seeds:

| 起手资源 | 速通(T) | Rusher 胜率 |
|----------|---------|-------------|
| 25(现状) | 5.1 | 0.0% |
| 21 | 11.4 | 12.2% |
| 19 | 11.7 | 10.2% |
| **18** | **30.5** | **54.2%** ← 势均力敌 |
| **17** | **30.3** | **53.2%** ← 势均力敌 |
| 16 | 44.2 | 86.5% |
| 14 | 45.0 | 86.2% |

## 结论

1. **一阶甜点存在,且在纯数值范围内**——不需要机制改动、不改科技树内容、不加游戏内容。
2. **关键杠杆是起手资源**(控制建设发育速度),不是设施门槛(无效)或科技耗时(天花板低)。
3. **甜点坐标 ≈ 起手资源 17-18**:建设速通 ~30T、Rusher vs Builder ~53%(势均力敌)。
4. **机制解释**(正是目标张力):资源少 → 建设者必须先发展经济才能负担 C 线科技 →
   "发育 vs 直接建设"取舍出现 → 建设慢到 30T → 军事 27T 攻城刚好够得着 → 势均力敌。

## M1.3:资源17 完整探针矩阵(400 seeds)

复现: `cargo run --release --bin eval -- 400 balanced out.json 17`

支配矩阵(行vs列 胜率%):
```
         Build Rush Haras Greed Evo Rand
 Builder   --   45   66   62   81   86
  Rusher   55   --   67   96   97   98
Harasser   34   33   --   84   88   96
  Greedy   38    4   16   --   79   46
     Evo   19    3   12   21   --   14
  Random   15    2    4   54   86   --
```

**关键结论(比"资源17=甜点"更 nuanced):**
1. **探针之间无单一支配**(维度A1 技术满足): 无任何探针对其他所有 >65%。
   Builder vs Rusher 45/55 互相制衡; Builder 克 Harasser(66); Rusher 克 Harasser(67)。有制衡雏形。
2. **胜利类型均衡了**: 征服53% / 建设37% / 平局9%(默认参数是建设压倒性)。两种胜利都大量出现。
3. **但手写 Greedy 被探针碾压**(维度B 失败): Rusher vs Greedy 96%, Harasser vs Greedy 84%。
   Greedy 平均37%。→ 这是 Greedy 反优化 + 不会防守, 不是设计问题, 留 M2 修。
4. **先手偏差**: Builder 镜像 P0=60%(资源少放大速通竞赛先手), 其余 ~52-54%, Rusher 32%(反常)。
   → 资源少的副作用, 需微调。

**判断**: 资源17 是很有希望的方向(胜利均衡+探针制衡), 但不是终点。两个待解:
- (a) 手写 AI 被碾压 → 需 M2 写会攻防兼顾/防守的 AI(纯调参无法让不设防的 Greedy 挡住 Rusher)。
- (b) Builder 镜像先手 60% → 需微调(资源略高 or 军事侧参数)。
- 应加 **Turtle(龟缩防守)探针** 检验"防守能否挡 Rusher"——当前缺这个维度。

**深层洞察**: "探针失能"不能纯靠调参达到——它和"AI 会不会应对"交织。
会防守的 AI 应能挡住 Rusher; 当前 Greedy/Random 不设防所以被屠。设计(参数)和 AI 能力必须一起调。
