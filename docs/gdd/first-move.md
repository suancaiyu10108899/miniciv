# GDD — 先手平衡

## 问题

旧项目 Random P0 = 67.6%（不可接受）。所有学习型方法被先手效应污染。

## 新设计的天然改善

| 旧设计 | 新设计 | 对先手的影响 |
|---|---|---|
| 固定角落开局 | 环面随机 + 等距偏移 | 双方拓扑等价 |
| 开局有战斗单位 | 只有工人+侦察兵 | 冷战争 10-15T → P0 边界优势被稀释 |
| 角落背靠墙 | 360° 暴露 | P0 不能"后背安全"单线推进 |
| 踩格得分 | 工人采集+项目系统 | 先手不能通过"先踩高分格"滚雪球 |

## 策略

**先不加任何补偿**。Python 原型完成后跑 Random vs Random N=500 在 balanced 30×30 上，看 baseline P0 winrate。

## 验证协议

```
1. Random vs Random, N=500, 30×30 balanced
   → 报告: P0_wr, P1_wr, side_advantage
   
2. 如果 P0 > 55%:
   a. 分析胜利路径分布（征服多？建设多？）
   b. 加方案 D（P1 初始城市周围额外 1-2 森林）
   c. 重跑 → 反复直到 P0 ≤ 55%
   
3. 在不同生成器上验证（symmetric/harsh/fertile）
   → 确保平衡不依赖单一生成器
```

## 候选补偿方案

| 方案 | 机制 | 触发条件 |
|---|---|---|
| D | P1 初始城市 3×3 外圈加 2 森林 | P0 > 55% 时 |
| A | P1 开局 +5 粮 | D 不够时叠加 |

目标：**Random P0 ≤ 55%**。不是 50%——55% 是"可接受"的合理范围。
